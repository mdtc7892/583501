# AI-Driven Intrusion Detection: A DDoS-Focused Framework for Real-Time Banking Network Security

Machine-learning pipeline that detects DDoS traffic in banking-style network flows using the
**CICIDS2017** dataset (Friday-Afternoon-DDoS capture). Three classifiers — Random Forest, SVM,
and a Neural Network (MLP) — are trained on SMOTE-balanced data and evaluated on accuracy,
precision, recall, F1, false-positive rate (FPR) and ROC-AUC. Predictions are made explainable
with **SHAP** (global feature importance) and **LIME** (per-prediction explanation).

This run uses the **full real dataset** (218,446 cleaned flows after filtering to BENIGN/DDoS,
removing identifier columns, and dropping inf/NaN/duplicate rows) — not the placeholder/sample
data the pipeline falls back to if no CSV is present.

## How to Run (VS Code / local)

1. Open the folder in VS Code, create a virtual environment, and install dependencies:
   ```bash
   python -m venv .venv
   source .venv/bin/activate        # Windows: .venv\Scripts\activate
   pip install -r requirements.txt
   ```
2. Make sure the raw CSV is in `dataset/` (already included in this package). If missing, see
   **Dataset** below.
3. Run the full pipeline from the project root:
   ```bash
   cd src
   python run_all.py
   ```
   Or run each stage individually, in order:
   ```bash
   python data_preprocessing.py
   python train_models.py
   python evaluate_models.py
   python explainability.py
   ```
4. Outputs land in `models/`, `output/figures/`, and `results/` automatically.

## Dataset

- **Source:** CICIDS2017, Canadian Institute for Cybersecurity (UNB) —
  https://www.unb.ca/cic/datasets/ids-2017.html
- **File used:** `Friday-WorkingHours-Afternoon-DDos.pcap_ISCX.csv`
- **Citation:** Sharafaldin, I., Lashkari, A.H., Ghorbani, A.A. (2018). *Toward Generating a New
  Intrusion Detection Dataset and Intrusion Traffic Characterization*. ICISSP.
- The raw CSV is included in `dataset/` for convenience, but is **excluded from git** via
  `.gitignore` (large-file hygiene). Re-download it from the link above, or from a public mirror,
  and drop it into `dataset/` before running the pipeline on a fresh clone.

## Results (full-dataset run)

| Model | Accuracy | Precision | Recall | F1-Score | FPR | ROC-AUC |
|---|---|---|---|---|---|---|
| Random Forest | 0.9996 | 1.0000 | 0.9993 | 0.9996 | 0.0000 | 1.0000 |
| SVM | 0.9908 | 0.9855 | 0.9989 | 0.9922 | 0.0205 | 0.9997 |
| Neural Network (MLP) | 0.9995 | 0.9999 | 0.9992 | 0.9995 | 0.0002 | 0.9999 |

Full table: `results/model_comparison.csv`. Charts: `output/figures/`.

**Note on SVM training size:** an RBF-kernel SVM does not scale to the full ~190K-row balanced
training set in a reasonable amount of time (kernel SVMs scale roughly O(n²)–O(n³)). SVM is
therefore fit on a stratified 20,000-row subsample of the balanced training data (see
`SVM_TRAIN_CAP` in `src/train_models.py`), while Random Forest and the MLP are trained on the
full balanced set. **All three models are evaluated on the same, full, untouched test set**, so
the comparison above remains fair and full-dataset-based on the evaluation side. This is a
standard, commonly documented practice when benchmarking kernel SVMs on large flow datasets —
call it out explicitly in the report/methodology section rather than presenting it as a full-data
SVM fit.
